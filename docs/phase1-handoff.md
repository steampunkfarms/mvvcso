## 2. CLAUDE.md Template

CC creates this as the very first action:

```markdown
# MVVCSO — CLAUDE.md

## Project
- **Name:** MVVCSO (Montezuma Valley Volunteer Community Service Organization)
- **Stack:** Next.js (App Router) + TypeScript + Tailwind v4 + Drizzle ORM + Neon Postgres
- **Hosting:** Vercel (Backcountry-Tech team, Pro)
- **Repo:** steampunkfarms/mvvcso
- **Domain:** mvvcso.vercel.app (production) → mvvcso.org later

## Conventions
- All env vars: `.trim()` before use
- All API routes: validate input with zod
- All cron routes: use `verifyCronAuth()` middleware
- Commit messages: conventional commits (feat:, fix:, chore:, docs:)
- No `any` types — strict TypeScript
- Prefer server components; use `'use client'` only when needed
- i18n: every user-facing string goes through the translation system from day 1
- Images: use `next/image` with proper width/height/alt
- Links: use `next/link`
- File naming: kebab-case for files, PascalCase for components
- CSS: Tailwind utility classes — no custom CSS files except globals.css for tokens

## Env Vars
- DATABASE_URL — Neon Postgres connection string
- BLOB_READ_WRITE_TOKEN — Vercel Blob storage
- RESEND_API_KEY — Email (newsletter, transactional)
- STRIPE_SECRET_KEY — Payments (server)
- STRIPE_PUBLISHABLE_KEY — Payments (client)
- STRIPE_WEBHOOK_SECRET — Stripe webhook verification
- NEXT_PUBLIC_SITE_URL — Base URL for absolute links
- ANTHROPIC_API_KEY — AI chatbot

## Brand: Rancheti Sunset Palette
See src/app/globals.css for CSS custom properties.
Primary: Sunset Gold #E8A838, Desert Coral #D85A3A
Secondary: Twilight Purple #6B4C8A, Sky Blue #4A90B8
Nature: Sage Brush #7A8B5E
Backgrounds: Sandstone #F5EDE0, Deep Sandstone #E8DCC8, Warm White #FAF7F2
Dark: Charcoal Earth #2D2420, Mountain Shadow #1A1216

## i18n
- English = default locale
- Spanish = co-primary (not secondary)
- All user-facing strings in translation files from day 1
- Spanish content can be placeholder text initially, but the STRUCTURE must support it

## Phase 1 Scope
Public site only. No admin dashboard.
Pages: Home, About/History, Programs, Blog, Donate, Contact, PCT
Newsletter signup, AI chatbot (placeholder OK), Stripe donations
```

## 5. Drizzle Schema (Phase 1 Only)

Only the tables needed for Phase 1 features. More tables in later phases.

```ts
// drizzle/schema.ts
import { pgTable, text, timestamp, uuid, boolean, integer } from 'drizzle-orm/pg-core';

// Newsletter subscribers
export const subscribers = pgTable('subscribers', {
  id: uuid('id').primaryKey().defaultRandom(),
  email: text('email').notNull().unique(),
  name: text('name'),
  language: text('language').notNull().default('en'), // 'en' | 'es'
  status: text('status').notNull().default('active'), // 'active' | 'unsubscribed'
  source: text('source').default('website'), // 'website' | 'footer' | 'floating-bar'
  createdAt: timestamp('created_at').notNull().defaultNow(),
  unsubscribedAt: timestamp('unsubscribed_at'),
});

// Contact form submissions
export const contactSubmissions = pgTable('contact_submissions', {
  id: uuid('id').primaryKey().defaultRandom(),
  name: text('name').notNull(),
  email: text('email').notNull(),
  phone: text('phone'),
  message: text('message').notNull(),
  language: text('language').notNull().default('en'),
  status: text('status').notNull().default('new'), // 'new' | 'read' | 'replied'
  createdAt: timestamp('created_at').notNull().defaultNow(),
});

// Donations (synced from Stripe webhooks)
export const donations = pgTable('donations', {
  id: uuid('id').primaryKey().defaultRandom(),
  stripeSessionId: text('stripe_session_id').unique(),
  stripeCustomerId: text('stripe_customer_id'),
  donorName: text('donor_name'),
  donorEmail: text('donor_email'),
  amount: integer('amount').notNull(), // cents
  currency: text('currency').notNull().default('usd'),
  type: text('type').notNull().default('one-time'), // 'one-time' | 'recurring'
  status: text('status').notNull().default('completed'),
  campaign: text('campaign').default('general'), // 'general' | 'ranchita-roots'
  createdAt: timestamp('created_at').notNull().defaultNow(),
});

// Blog posts (stored in DB for now; can migrate to MDX files later)
export const blogPosts = pgTable('blog_posts', {
  id: uuid('id').primaryKey().defaultRandom(),
  slug: text('slug').notNull().unique(),
  titleEn: text('title_en').notNull(),
  titleEs: text('title_es'),
  contentEn: text('content_en').notNull(), // Markdown
  contentEs: text('content_es'), // Markdown
  excerpt_en: text('excerpt_en'),
  excerpt_es: text('excerpt_es'),
  author: text('author').default('MVVCSO'),
  category: text('category'), // 'community' | 'programs' | 'history' | 'resources'
  coverImage: text('cover_image'),
  published: boolean('published').notNull().default(false),
  publishedAt: timestamp('published_at'),
  createdAt: timestamp('created_at').notNull().defaultNow(),
  updatedAt: timestamp('updated_at').notNull().defaultNow(),
});
```

## 7. Route-by-Route Page Specs

### 7.1 Homepage (`/`)

**Goal:** Make MVVCSO look world-class on first visit. Emotional, visual, professional.

**Sections (top to bottom):**

1. **Hero** — Full-bleed background image of Ranchita high desert landscape. Large mission statement text overlaid. Two CTA buttons: "Donate" (sunset gold) + "Volunteer" (outlined). Subtle parallax or Ken Burns effect on the image. Mobile: stack buttons, reduce text size.

2. **Mission / About preview** — Short paragraph with the grant-optimized mission statement (from Drive doc). Link to full About page. Maybe a small Rancheti illustration as a decorative element.

3. **Programs grid** — 4-6 cards showing key programs: CERT, Food Pantry, Youth Programs, Senior Support, Emergency Preparedness, Volunteer Fire History. Each card: icon/image, title, 1-line description, "Learn more" link.

4. **Impact stats** — Animated counter section (USE INTERSECTION OBSERVER — do NOT repeat the BTS counter bug). Stats: "50+ Years of Service", "300+ Residents Served", "7 Board Members", "100% Volunteer-Run". Numbers count up when scrolled into view. Dark background (charcoal-earth) for contrast.

5. **Partners bar** — Horizontal logo strip: Feeding San Diego + other partners. Light gray background.

6. **Newsletter CTA** — "Get the Ranchita Resilience Report" — email input + subscribe button. Sunset gold accent. Bilingual placeholder text.

7. **Donate CTA** — Final section before footer. Emotional appeal. "Support Your Community" with impact framing. Link to /donate.

### 7.2 About / History (`/about`)

**Goal:** Tell the MVVCSO story compellingly. Radical transparency.

**Sections:**

1. **Hero** — Page title "Our Story" with a subtler hero image.

2. **Interactive Timeline** — The crown jewel. Scroll-triggered vertical timeline covering:
   - 1973: MVVFD founded
   - 1977: Incorporated as nonprofit
   - 1980s–90s: Growth, community BBQs, multigenerational service
   - 2002: Pines Fire response
   - 2003: Cedar Fire response
   - 2004: FEMA grant ($41,500)
   - 2008: Integration into County Fire Authority / Station 58
   - 2013–14: New fire station built ($880K)
   - 2020: Rebrand to MVVCSO
   - 2021: Montezuma Valley Market fire response
   - Present: Community programs, Community Center campaign

   Each timeline node: year, title, 2-3 sentence description, optional photo.
   Animate in on scroll. Use intersection observer.

3. **Board of Directors** — Grid of 7 board members. Name + role. No photos needed initially (placeholder silhouettes). Source: Annette Foote, Colleen James, David Walter, Gabby Ohmert, Kristi Bruner, Michelle Erwin, Nicholas Ketelesen.

4. **Partners** — Expanded partner section with descriptions.

5. **Transparency note** — Short statement about radical transparency, link to bylaws/minutes when available.

### 7.3 Programs (`/programs`)

**Sections:**

1. Overview paragraph about MVVCSO's community service mission.

2. Program cards (expandable or linked to detail sections):
   - **Community Emergency Response Team (CERT)** — Training, preparedness
   - **Food Security** — Feeding San Diego partnership, CalFresh enrollment
   - **Senior Support** — Services for elderly residents
   - **Youth Programs** — Ranchita Youth Council, education, mentorship
   - **Emergency Preparedness** — Disaster readiness, communications
   - **Mobile Health Clinics** — Healthcare access for rural residents
   - **Volunteer Services** — General volunteer coordination

3. CTA: "Want to get involved? Volunteer today."

### 7.4 Blog (`/blog` + `/blog/[slug]`)

**Blog index:** Grid of post cards with cover image, title, excerpt, date, category badge.

**Blog post:** Clean reading layout. Markdown rendered to HTML. Author, date, category. Share buttons. Related posts at bottom.

**Initial seed posts (CC can write these or use placeholder content):**
1. "Welcome to the New MVVCSO Website"
2. "A History of Service: From Volunteer Firefighters to Community Champions"
3. "How to Prepare for Fire Season in the Backcountry"
4. "Getting Connected in Ranchita: Internet and Communications Guide"

### 7.5 Donate (`/donate`)

**Sections:**

1. **Hero** — Emotional appeal. "Every dollar builds a stronger Ranchita."

2. **Donation form** — Stripe Checkout or Stripe Elements. Preset amounts: $25, $50, $100, $250, Custom. Toggle: One-time / Monthly recurring.

3. **Campaign selector** — "General Fund" or "Ranchita Roots" (community center campaign — hold back for now, but build the dropdown).

4. **Impact framing** — "$25 = emergency supply kit", "$50 = feeds 10 families", "$100 = CERT training for 1 volunteer", "$250 = supports youth program for a month".

5. **Tax deductible notice** — "MVVCSO is a 501(c)(3) nonprofit. Your donation is tax-deductible."

6. **Secondary option** — Link to existing Zeffy page for those who prefer it.

**API:** `POST /api/donate/create-session` → Creates Stripe Checkout session → Redirects. Webhook at `/api/donate/webhook` handles `checkout.session.completed` → writes to `donations` table.

### 7.6 Contact (`/contact`)

**Sections:**

1. **Contact form** — Name, Email, Phone (optional), Message, Language preference. Submit → writes to `contact_submissions` table + sends email via Resend to board.

2. **Contact info** — Address: 37370 Montezuma Valley Rd, Ranchita, CA 92066. Phone: (760) 782-8449. Email: info@mvvcso.org (or ranchitacommunityorganization@gmail.com for now).

3. **Map** — Google Maps or Mapbox embed showing Ranchita / Station 58 location.

4. **Hours** — If applicable, community center hours or board meeting schedule.

### 7.7 Virtual PCT Hike (`/pct`)

**Goal:** Unique, shareable feature. Interactive experience of the Pacific Crest Trail through Ranchita's backyard.

**Implementation:**

1. **Interactive map** (Leaflet with OpenStreetMap or Mapbox) showing the PCT segment through Montezuma Valley / Ranchita area. Trail line highlighted.

2. **Waypoints** — Clickable markers at scenic points. Each opens a panel with:
   - Photo (web-sourced initially)
   - Elevation
   - Description of what you see
   - Historical context where applicable
   - Flora/fauna notes

3. **Elevation profile** — SVG chart below the map showing elevation changes along the segment.

4. **Community context** — Sidebar or below-map section: "The PCT passes through our community. Ranchita has served as a resupply point for thru-hikers for decades."

5. **Links** — AllTrails, official PCT website, trail conditions.

This can be simpler for launch — a beautiful map with 5-8 waypoints and photos. Expandable later.

// src/lib/env.ts
function getEnv(key: string, required = true): string {
  const value = process.env[key]?.trim();
  if (!value && required) {
    console.warn(`⚠️ Missing env var: ${key}`);
    return '';
  }
  return value ?? '';
}

## 15. Important Reminders for CC

1. **INTERSECTION OBSERVER** for counter animations. The BTS site had a bug where counters flashed "0" on load — use IntersectionObserver to trigger animation only when visible.

2. **No hardcoded English strings** in components. Every user-facing string goes through `next-intl`'s `useTranslations()` hook or `getTranslations()` server function.

3. **Graceful API key fallbacks.** If Stripe key is missing, show donate page with a "Donation processing coming soon" message + link to Zeffy. If Resend key is missing, log the subscription to DB only. If Anthropic key is missing, show static chatbot message.

4. **Mobile-first.** Design for 375px first, then scale up. Ranchita residents are on phones over Starlink.

5. **Code-split aggressively.** The PCT map page with Leaflet should be dynamic-imported. Blog post markdown rendering should be lazy. Every below-fold component should use intersection observer for lazy loading.

6. **Write checkpoints.** After each sub-phase (1a, 1b, 1c, 1d), write a checkpoint to `docs/checkpoints/`. If you sense a timeout coming, write one immediately.

7. **Don't over-engineer.** Phase 1 is the public site. No auth, no admin, no database-heavy features beyond newsletter/contact/donations. Keep it simple and beautiful.
